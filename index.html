Applications of Quest Hand Tracking: VR Piano Keyboard

Progress Report

Brian Lau, Jason Yih, Suruchi Walekar

Initial Goals

Our initial goal was to make a functional VR piano keyboard that would be as close as possible to the physics and interactions of a real keyboard to test the limits of hand tracking. Our motivation was that a VR keyboard would allow people to test chords and learn basic songs with only a VR headset. By April 30th, we wanted to achieve a working segment of a keyboard with accurate finger tracking, key presses, interactions, and sounds. By May 15th, we want to take velocity and position into account when interacting with keys, expand to a full keyboard, improve physics, and add static sheet music.

Current Progress

As of now, we are on track in terms of our goals for our MVP, with a working segment of a keyboard that allows for pressing the keys using hand tracking. Basic sound generation and Rigidbody interactions have been implemented, as well as initial velocity-based key pressing.

  • Suruchi
  • Set up basic hand tracking using the built in OVRSkeleton, adding a visible mesh so that the user can see their hands in virtual space, and observing the accuracy of hand tracking as-is
  • Added Rigidbody physics to both the piano keys and the hands for more realistic interactions, and started tracking finger velocity to measure presses
  • Tested changing spring force and strength of presses for greater accuracy
  • Fixed initial errors such as floating and unpredictable key positions
  • Brian
  • Added an initial keyboard segment and table with colliders for easier testing, as well as code for working keys that generate sound
  • Implemented volume control so keys are louder proportional to the strength of a press
  • Added fadeouts so that notes linger slightly when they are pressed, similar to a real piano
  • Improved key return and damping as well as issues with clamping so keys would return faster after being pressed and the visual representations would be more stable
  • Jason
  • Improved finger tracking and piano responsiveness
  • Implemented use of segments of fingers instead of fingertips to allow for accurate key presses
  • Fixed bugs such as keys playing over and over, as well as issues with no sound when keys were pressed

Remaining Goals

Our remaining goals include making the black keys functional by editing the current meshes and how they are placed, continuing to improve upon physical interactions and sound generation, adding dynamic keyboard and table positioning, expanding to a full keyboard with more accurately sized keys, and adding static sheet music. In case we encounter time constraints, sheet music can be moved to long-term goals and future work in order for us to focus on creating the most realistic physics possible for the best user experience.

Expected Contributions

Our expected contributions remain largely the same as our initial project proposal. We aim to complete all or most of the remaining goals, with the order of priority being to ensure all keys are functional, improve the tracking of individual fingers, having a full keyboard, and reducing the key size as much as hand tracking will allow, to create a fully functional MVP.

Long-Term Goals

If we are able to add sample sheet music and improve the general aesthetics of the app, our long-term goal is to register as developers on the Meta Store or App Lab to bring our app to more users for feedback and ideas for improvement.
